# AI Hub

Якщо вам потрібен окремий настільний застосунок із додатковими сценаріями роботи, перегляньте [BasiliskLLM](https://github.com/SigmaNight/basiliskLLM/) (самостійний застосунок і мінімальний додаток NVDA). AI Hub залишається повнофункціональним рішенням усередині NVDA.

**AI Hub** — це додаток NVDA, який підключає екранний читач до кількох API великих мовних моделей (LLM). Він дає змогу писати, підсумовувати, перекладати, описувати зображення (включно зі скриншотами), ставити голосові запитання, робити транскрипцію та користуватися додатковими інструментами (TTS, OCR тощо) без виходу з NVDA.

**Назва пакета** в NVDA і далі `openai` (для сумісності з наявними встановленнями). **Відображувана назва** в меню та налаштуваннях — **AI Hub**.

## Можливості

- **Чат** в окремому головному діалозі з історією, системним запитом та вибором моделі/акаунта.
- **Зображення та документи** як вкладення з файлів; **URL** віддалених файлів із перевіркою типів відповідно до **вибраного провайдера**.
- **Розумна вставка** у полі запиту: вставлення файлів із буфера обміну, шляхів із тексту або одного URL (також через контекстне меню поля запиту). `Ctrl+V` працює за тією самою логікою, коли фокус у полі запиту.
- **Історія розмов**: збереження, перейменування, видалення та повторне відкриття.
- **Поставити запитання** з будь-якого місця (без жесту за замовчуванням): призначте жест у **Жести вводу -> AI Hub**, щоб записати, надіслати й прослухати/прочитати відповідь.
- **Глобовий опис**: скриншот (`NVDA+E`) або область об'єкта навігатора (`NVDA+O`) надсилається в чат-сесію.
- Підменю **Інструменти** (NVDA -> AI Hub): утиліти для конкретних провайдерів, зокрема TTS, OCR, speech-to-text, Lyria audio і керування моделями Ollama.
- Опції **reasoning / web search** показуються лише тоді, коли **поточна модель** їх підтримує.

У цьому додатку **немає** власного перевіряча оновлень. **Оновлення** встановлюються через **офіційний Магазин додатків NVDA**, якщо ви інсталювали його звідти. Для ручного встановлення з [сторінки релізів](https://github.com/aaclause/nvda-OpenAI/releases) інсталюйте нові `.nvda-addon` аналогічно.

## Підтримувані провайдери

Налаштуйте **одного або кількох провайдерів** у NVDA: **Параметри -> Налаштування -> AI Hub**. Для кожного провайдера можна зберігати **кілька іменованих акаунтів** (API-ключі, а за потреби organization/base URL).

| Провайдер | Роль |
|----------|------|
| [OpenAI](https://platform.openai.com/) | GPT і пов'язані моделі; офіційні діалоги транскрипції та TTS |
| [DeepSeek](https://www.deepseek.com/) | API DeepSeek (сумісний з OpenAI) |
| **Custom OpenAI** | Будь-який HTTP API, сумісний з OpenAI (власний base URL + ключ) |
| **Ollama** | Локальні моделі через endpoint, сумісний з OpenAI; менеджер моделей |
| [Mistral AI](https://mistral.ai/) | Mistral / Pixtral; інструменти Voxtral TTS, OCR і speech-to-text |
| [OpenRouter](https://openrouter.ai/) | Багато сторонніх моделей через один ключ |
| [Anthropic](https://www.anthropic.com/) | Claude |
| [xAI](https://x.ai/) | Grok |
| [Google](https://ai.google.dev/) | Gemini; інструмент Lyria 3 Pro |

Додаток може підхоплювати API-ключі з **змінних середовища** (наприклад, `OPENAI_API_KEY`, `DEEPSEEK_API_KEY`, `MISTRAL_API_KEY`, `OPENROUTER_API_KEY`, `ANTHROPIC_API_KEY`, `XAI_API_KEY`, `GEMINI_API_KEY` / `GOOGLE_API_KEY` та інші). Основний спосіб керування акаунтами — інтерфейс налаштувань.

### Бекенди розпізнавання мовлення (транскрипції)

Для **транскрипції з мікрофона/файлу** в основному потоці (не окремий інструмент OpenAI) можна обрати **whisper_cpp** (локально), **openai** (Whisper API) або **mistral** у розділі **Audio** налаштувань AI Hub.

## Встановлення

1. Відкрийте [сторінку релізів додатка](https://github.com/aaclause/nvda-OpenAI/releases).
2. Завантажте актуальний `.nvda-addon`.
3. Встановіть через **Інструменти -> Керування додатками** у NVDA (або відкрийте файл у Провіднику й підтвердьте встановлення).

## Початкове налаштування

1. Відкрийте **NVDA -> Параметри -> Налаштування**.
2. Виберіть категорію **AI Hub**.
3. У розділі **API keys** скористайтеся кнопками потрібного провайдера (наприклад, **OpenAI API keys...**) і додайте щонайменше один акаунт: назву, API-ключ і (за потреби) додаткові поля.
4. За потреби налаштуйте **Audio**, **Chat feedback**, **Advanced** / temperature і **Auto-save conversation** (типово увімкнено).

Поки не налаштовано хоча б один робочий акаунт провайдера, під час відкриття головного вікна з'являтиметься пропозиція додати ключі в налаштуваннях AI Hub.

## Оновлення зі старих версій "Open AI"

Якщо ви використовували старішу версію:

- **Налаштування** мігрують зі старого розділу **`OpenAI`** до **`AIHub`**.
- **Дані** (розмови, сховище ключів, вкладення) мігрують з папки **`openai`** до **`aihub`** у каталозі користувацької конфігурації NVDA.

Зазвичай переносити файли вручну не потрібно.

## Меню NVDA: AI Hub

У меню NVDA ви знайдете **AI Hub** (із номером версії в назві). Пункти містять:

- **Documentation** — відкриває документацію в браузері (`doc\en\readme.html`). У релізному пакеті цей файл має бути включений; його **не** редагують вручну, а генерують із `readme.md` під час складання.
- **Main dialog...** — відкрити вікно чату (`NVDA+G` типово).
- **Conversation history...** — керування збереженими чатами.
- **Tools** — підменю інструментів **OpenAI**, **Mistral**, **Google** та **Ollama**.
- **API keys** / **API usage** / **GitHub repository** / **BasiliskLLM** — швидкі посилання.

## Головний діалог

Відкривається через **`NVDA+G`** або пункт **Main dialog...** у меню AI Hub.

### Що можна робити

- Спілкуватися з вибраною моделлю; навігація по **Messages** з клавіатури та контекстного меню (наприклад, **j** / **k** у полі повідомлень).
- Додавати **локальні зображення/документи** та **URL файлів**, якщо це підтримує провайдер.
- Використовувати **Paste (file or text)** з контекстного меню поля запиту або **`Ctrl+V`**: додаток може прикріпити файл, вставити шлях або обробити одиночний URL як вкладення.
- Записувати **аудіо**, додавати аудіофайли та використовувати **TTS** для тексту запиту, якщо модель це підтримує.
- **`Escape`** закриває головний діалог (якщо немає блокувального модального вікна).
- **`Ctrl+R`** запускає/зупиняє запис із мікрофона.
- **`F2`** перейменовує поточну збережену розмову.
- **`Ctrl+N`** відкриває новий сеанс у новому вікні.

### Опції, що залежать від моделі

Деякі елементи доступні лише для окремих моделей:

- **Reasoning** ("thinking"), якщо модель підтримує його.
- **Reasoning effort** та пов'язані параметри, якщо провайдер підтримує рівні.
- **Web search** лише для моделей, які заявляють підтримку веб-пошуку.

Точна доступність змінюється разом з API провайдерів; інтерфейс відображає стан для **поточної моделі**.

### Системний запит

Системний запит керує поведінкою моделі. Є типовий запит, орієнтований на доступність; ви можете редагувати його, скидати з контекстного меню та (за бажання) зберігати останній використаний варіант. Системний запит теж споживає токени.

## Історія розмов

Скористайтеся **Conversation history...** у меню AI Hub або призначте жест у **Жести вводу -> AI Hub**.

Можна переглядати, відкривати, перейменовувати, видаляти та створювати розмови. У головному діалозі для цього зручно використовувати **F2** і **Ctrl+N**.

### Автозбереження

Якщо **Auto-save conversation** увімкнено (типово), розмова зберігається/оновлюється **після кожної завершеної відповіді помічника**. Також стан може бути збережено під час закриття діалогу. Є і ручне збереження з контекстного меню поля **Messages**.

## Поставити запитання (голосом)

Ця команда **не має** жесту за замовчуванням. Призначте його в **Жести вводу -> AI Hub**.

- Перше натискання: почати запис.
- Друге натискання під час запису: зупинити й надіслати.
- Якщо відповідь відтворюється голосом, наступне натискання зупинить відтворення.

**Режими:**

- **Direct audio** — якщо поточна модель підтримує аудіовхід, запис надсилається напряму.
- **Transcribe then chat** — інакше запис спочатку транскрибується, а вже потім текст надсилається в чат-модель.

Якщо головний діалог у фокусі, використовується його **поточна модель**; інакше додаток обирає відповідну модель автоматично.

## Підменю Tools

Пункт **Tools** відкриває діалоги, згруповані за провайдерами (кожен може вимагати відповідний API-акаунт):

| Розділ меню | Інструмент |
|-----------|------|
| Mistral | **Voxtral TTS...**, **OCR...**, **Speech to Text...** |
| Google | **Lyria 3 Pro...** |
| OpenAI | **TTS...**, **Transcription / Translation...** |
| Ollama | **Model manager...** |

Якщо акаунт потрібного провайдера не налаштовано, додаток запропонує додати його в налаштуваннях AI Hub.

## Глобальні команди

Усі жести можна змінити в **NVDA -> Параметри -> Жести вводу -> AI Hub**.

| Жест | Дія |
|---------|--------|
| `NVDA+G` | Показати головний діалог AI Hub |
| `NVDA+E` | Зробити скриншот і описати |
| `NVDA+O` | Описати область поточного об'єкта навігатора |
| *(без жесту за замовчуванням)* | Історія розмов |
| *(без жесту за замовчуванням)* | Поставити запитання (запис / надсилання / зупинка аудіо) |
| *(без жесту за замовчуванням)* | Увімкнути/вимкнути запис з мікрофона і транскрипцію |

## Де зберігаються дані

Робочі файли, індекс збережених розмов, `accounts.json` і вкладення зберігаються в папці **`aihub`** у користувацькій конфігурації NVDA (після міграції з `openai`). Тимчасові файли розміщуються у `tmp` і очищуються за потреби.

## Потрібні залежності (автозавантаження під час складання)

Складання через `scons` готує runtime-бібліотеки в:

`addon/globalPlugins/AIHub/libs/`

Якщо потрібної бібліотеки немає, `scons` завантажує зафіксовані (pinned) wheel-пакети та витягає тільки необхідне. Поточні залежності:

- **[markdown2](https://pypi.org/project/markdown2/)** `2.5.4` — витягується як `libs/markdown2.py` для рендерингу Markdown у чаті.
- **[Pillow](https://pypi.org/project/Pillow/)** `12.1.1` для:
  - Python `3.11` `win32` -> `libs/lib_py3.11_win32/`
  - Python `3.13` `win_amd64` -> `libs/lib_py3.13/`

Папка `libs` навмисно ігнорується git; ці артефакти не потрібно комітити.

## Коротке усунення несправностей

- **"No account configured"** — додайте API-ключ для вибраного провайдера в налаштуваннях **AI Hub**.
- **Провайдер відхиляє вкладення** — перевірте тип і розмір файлу; за потреби спробуйте іншу модель або провайдера.

Для повідомлень про проблеми та внесків використовуйте посилання **GitHub repository** в меню AI Hub.
