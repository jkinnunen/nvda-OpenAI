# AI Hub

Если вам нужен отдельный настольный клиент с дополнительными сценариями работы, посмотрите [BasiliskLLM](https://github.com/SigmaNight/basiliskLLM/) (самостоятельное приложение и минимальный аддон NVDA). AI Hub остается полнофункциональным вариантом внутри NVDA.

**AI Hub** — это дополнение NVDA, которое подключает экранный диктор к нескольким API больших языковых моделей (LLM). Оно позволяет писать и редактировать тексты, переводить, кратко пересказывать, описывать изображения (включая скриншоты), задавать голосовые вопросы, выполнять транскрибацию и использовать дополнительные инструменты (TTS, OCR и др.) не выходя из NVDA.

**Имя пакета** в NVDA по-прежнему `openai` (для совместимости с существующими установками). **Отображаемое имя** в меню и настройках — **AI Hub**.

## Кратко о возможностях

- **Чат** в отдельном основном окне с историей, системным промптом и выбором модели/аккаунта.
- **Изображения и документы** как вложения из файлов; **URL** удаленных файлов с проверкой типов в соответствии с **выбранным провайдером**.
- **Умная вставка** в поле промпта: вставка файлов из буфера обмена, путей из текста или одного URL (также через контекстное меню поля). `Ctrl+V` использует ту же логику, когда фокус в поле ввода.
- **История разговоров**: сохранение, переименование, удаление и повторное открытие.
- **Задать вопрос** из любого места (без жеста по умолчанию): назначьте жест в **Жесты ввода -> AI Hub**, чтобы записать вопрос, отправить его и получить ответ.
- **Глобальное описание**: скриншот (`NVDA+E`) или область объекта навигатора (`NVDA+O`) отправляется в чат-сессию.
- Подменю **Инструменты** (NVDA -> AI Hub): утилиты для конкретных провайдеров, например TTS, OCR, speech-to-text, Lyria audio и управление моделями Ollama.
- Опции **reasoning / web search** показываются только для **текущей модели**, если она их поддерживает.

В дополнении **нет** собственного механизма обновлений. **Обновления** ставятся через **официальный Магазин дополнений NVDA**, если вы установили его оттуда. При ручной установке с [страницы релизов](https://github.com/aaclause/nvda-OpenAI/releases) устанавливайте новые `.nvda-addon` аналогично.

## Поддерживаемые провайдеры

Настройте **одного или нескольких провайдеров** в NVDA: **Параметры -> Настройки -> AI Hub**. Для каждого провайдера можно хранить **несколько именованных аккаунтов** (API-ключ, а также при необходимости organization/base URL).

| Провайдер | Роль |
|----------|------|
| [OpenAI](https://platform.openai.com/) | GPT и связанные модели; официальные диалоги транскрибации и TTS |
| [DeepSeek](https://www.deepseek.com/) | API DeepSeek (совместимый с OpenAI) |
| **Custom OpenAI** | Любой OpenAI-совместимый HTTP API (свой base URL + ключ) |
| **Ollama** | Локальные модели через OpenAI-совместимый endpoint; менеджер моделей |
| [Mistral AI](https://mistral.ai/) | Mistral / Pixtral; инструменты Voxtral TTS, OCR и speech-to-text |
| [OpenRouter](https://openrouter.ai/) | Доступ ко многим моделям через один ключ |
| [Anthropic](https://www.anthropic.com/) | Claude |
| [xAI](https://x.ai/) | Grok |
| [Google](https://ai.google.dev/) | Gemini; инструмент Lyria 3 Pro |

Дополнение может считывать API-ключи из **переменных окружения** (например, `OPENAI_API_KEY`, `DEEPSEEK_API_KEY`, `MISTRAL_API_KEY`, `OPENROUTER_API_KEY`, `ANTHROPIC_API_KEY`, `XAI_API_KEY`, `GEMINI_API_KEY` / `GOOGLE_API_KEY` и другие). Основной способ управления аккаунтами — интерфейс настроек.

### Бэкенды распознавания речи (транскрибации)

Для **транскрибации с микрофона/файла** в основном сценарии (не отдельный инструмент OpenAI) можно выбрать **whisper_cpp** (локально), **openai** (Whisper API) или **mistral** в разделе **Audio** настроек AI Hub.

## Установка

1. Откройте [страницу релизов дополнения](https://github.com/aaclause/nvda-OpenAI/releases).
2. Скачайте актуальный `.nvda-addon`.
3. Установите через **Инструменты -> Управление дополнениями** в NVDA (или откройте файл из Проводника и подтвердите установку).

## Первоначальная настройка

1. Откройте **NVDA -> Параметры -> Настройки**.
2. Выберите категорию **AI Hub**.
3. В разделе **API keys** нажмите кнопку нужного провайдера (например, **OpenAI API keys...**) и добавьте хотя бы один аккаунт: отображаемое имя, API-ключ и (при необходимости) organization/base URL.
4. При необходимости настройте **Audio**, **Chat feedback**, **Advanced** / temperature и **Auto-save conversation** (по умолчанию включено).

Пока не настроен хотя бы один рабочий аккаунт, при открытии основного окна появится предложение добавить ключи в настройках AI Hub.

## Обновление со старых версий "Open AI"

Если вы использовали более старую версию:

- **Настройки** мигрируются из старого раздела **`OpenAI`** в **`AIHub`**.
- **Данные** (разговоры, хранилище ключей, вложения) мигрируются из папки **`openai`** в **`aihub`** внутри каталога пользовательской конфигурации NVDA.

Обычно вручную переносить файлы не требуется.

## Меню NVDA: AI Hub

В меню NVDA появится раздел **AI Hub** (с номером версии в подписи). Пункты включают:

- **Documentation** — открывает руководство в браузере (`doc\en\readme.html`). В опубликованном пакете этот файл должен быть включен; он **не** редактируется вручную, а генерируется из `readme.md` на этапе сборки.
- **Main dialog...** — открыть окно чата (`NVDA+G` по умолчанию).
- **Conversation history...** — управление сохраненными чатами.
- **Tools** — подменю инструментов **OpenAI**, **Mistral**, **Google** и **Ollama**.
- **API keys** / **API usage** / **GitHub repository** / **BasiliskLLM** — быстрые ссылки.

## Главное окно

Открывается через **`NVDA+G`** или пункт **Main dialog...** в меню AI Hub.

### Что можно делать

- Общаться с выбранной моделью; перемещаться по **Messages** с клавиатуры и через контекстные меню (например, **j** / **k** в области сообщений).
- Прикреплять **локальные изображения/документы** и добавлять **URL файлов** (если поддерживается провайдером).
- Использовать **Paste (file or text)** в контекстном меню поля ввода или **`Ctrl+V`**: дополнение может прикреплять файл, вставлять путь или трактовать единичный URL как вложение.
- Записывать **аудио**, добавлять аудиофайлы и использовать **TTS** для текста промпта (если поддерживается).
- **`Escape`** закрывает окно (если нет блокирующего модального диалога).
- **`Ctrl+R`** запускает/останавливает запись с микрофона.
- **`F2`** переименовывает текущую сохраненную беседу.
- **`Ctrl+N`** открывает **новое** окно сессии.

### Опции, зависящие от модели

Некоторые элементы доступны только для определенных моделей:

- **Reasoning** ("thinking"), если модель поддерживает отдельный поток рассуждений.
- **Reasoning effort** и связанные настройки, если провайдер поддерживает уровни.
- **Web search** только для моделей, которые заявляют поддержку веб-поиска.

Точный набор опций зависит от выбранной модели и может меняться вместе с API провайдеров.

### Системный промпт

Системный промпт определяет поведение модели. Есть готовый промпт, ориентированный на задачи доступности; его можно редактировать, сбрасывать из контекстного меню и (опционально) сохранять последний использованный вариант. Системный промпт также расходует токены.

## История разговоров

Используйте **Conversation history...** в меню AI Hub или назначьте жест в **Жесты ввода -> AI Hub**.

Можно просматривать, открывать, переименовывать, удалять и создавать разговоры. В основном окне помогают **F2** и **Ctrl+N**.

### Автосохранение

Если **Auto-save conversation** включено (по умолчанию), разговор сохраняется/обновляется **после каждого завершенного ответа ассистента**. Дополнительно состояние может сохраняться при закрытии окна. Также доступно ручное сохранение из контекстного меню области **Messages**.

## Задать вопрос (голосом)

Эта команда **не имеет жеста по умолчанию**. Назначьте его в **Жесты ввода -> AI Hub**.

- Первое нажатие: начать запись.
- Второе нажатие во время записи: остановить и отправить.
- Если ответ проигрывается как аудио, следующее нажатие остановит воспроизведение.

**Режимы:**

- **Direct audio** — если модель принимает аудио напрямую, запись отправляется без отдельной транскрибации.
- **Transcribe then chat** — иначе запись сначала распознается выбранным бэкендом, после чего текст отправляется модели чата.

Если активно главное окно, используется его **текущая модель**; иначе дополнение выберет подходящую модель из настроенных провайдеров.

## Подменю Tools

Пункт **Tools** открывает диалоги по провайдерам (для каждого может требоваться соответствующий аккаунт API):

| Раздел меню | Инструмент |
|-----------|------|
| Mistral | **Voxtral TTS...**, **OCR...**, **Speech to Text...** |
| Google | **Lyria 3 Pro...** |
| OpenAI | **TTS...**, **Transcription / Translation...** |
| Ollama | **Model manager...** |

Если аккаунт нужного провайдера не настроен, дополнение подскажет добавить его в настройках AI Hub.

## Глобальные команды

Все жесты можно переназначить в **NVDA -> Параметры -> Жесты ввода -> AI Hub**.

| Жест | Действие |
|---------|--------|
| `NVDA+G` | Показать главное окно AI Hub |
| `NVDA+E` | Сделать скриншот и описать |
| `NVDA+O` | Описать область текущего объекта навигатора |
| *(без жеста по умолчанию)* | История разговоров |
| *(без жеста по умолчанию)* | Задать вопрос (запись / отправка / остановка аудио) |
| *(без жеста по умолчанию)* | Запуск/остановка записи и транскрибации микрофона |

## Где хранятся данные

Рабочие файлы, индекс сохраненных разговоров, `accounts.json` и вложения хранятся в папке **`aihub`** в каталоге пользовательской конфигурации NVDA (после миграции из `openai`). Временные файлы находятся в `tmp` и очищаются при необходимости.

## Зависимости (автозагрузка при сборке)

Сборка через `scons` подготавливает runtime-библиотеки в:

`addon/globalPlugins/AIHub/libs/`

Если требуемой библиотеки нет, `scons` скачивает зафиксированные (pinned) wheel-пакеты и извлекает только нужные файлы. Текущие зависимости:

- **[markdown2](https://pypi.org/project/markdown2/)** `2.5.4` — извлекается как `libs/markdown2.py` (рендеринг Markdown в чате).
- **[Pillow](https://pypi.org/project/Pillow/)** `12.1.1` для:
  - Python `3.11` `win32` -> `libs/lib_py3.11_win32/`
  - Python `3.13` `win_amd64` -> `libs/lib_py3.13/`

Каталог `libs` намеренно игнорируется в git; его не нужно коммитить.

## Коротко о проблемах

- **"No account configured"** — добавьте API-ключ выбранного провайдера в настройках **AI Hub**.
- **Провайдер отклоняет вложение** — проверьте тип и размер файла; при необходимости попробуйте другой провайдер/модель.

Для сообщений об ошибках и вкладов используйте ссылку **GitHub repository** в меню AI Hub.
