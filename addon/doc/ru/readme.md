Если вам нужен отдельный настольный клиент с дополнительными сценариями работы, посмотрите [BasiliskLLM](https://github.com/SigmaNight/basiliskLLM/) (самостоятельное приложение и минимальный аддон NVDA). AI Hub остается полнофункциональным вариантом внутри NVDA.

# AI Hub

**AI Hub** — это дополнение NVDA, которое подключает ваш экранный диктор к нескольким API больших языковых моделей (LLM). Используйте его для написания текстов, резюмирования, помощи с переводом, работы с изображениями (включая картинки и скриншоты), голосовых вопросов, транскрибации и дополнительных инструментов (TTS, OCR и др.) — не выходя из NVDA.

**Имя пакета** дополнения в NVDA по-прежнему `openai` (для совместимости с существующими установками). **Отображаемое имя**, которое вы видите в меню и настройках, — **AI Hub**.

## Кратко о возможностях

- **Чат** в отдельном главном диалоге с историей, системным промптом и выбором модели/аккаунта.
- **Изображения и документы** как вложения из файлов; **URL** удаленных файлов с проверкой типов в соответствии с **выбранным провайдером**.
- **Умная вставка** в поле промпта: вставка файлов из буфера обмена, путей из текста или одного URL (также доступно из контекстного меню поля). `Ctrl+V` использует ту же логику, когда фокус находится в поле промпта.
- **Сохранение разговоров и история** с переименованием, удалением и повторным открытием.
- **Задать вопрос** из любого места (без жеста по умолчанию): назначьте жест в **Жесты ввода -> AI Hub**, чтобы записать, отправить, а затем прослушать или прочитать ответ.
- **Глобальное описание**: скриншот (`NVDA+E`) или область объекта навигатора (`NVDA+O`) отправляются в чат-сессию.
- Подменю **Tools** (в NVDA -> AI Hub): утилиты для конкретных провайдеров, такие как TTS, OCR, speech-to-text, Lyria audio и управление моделями Ollama.
- Параметры **Reasoning / Web search** появляются только когда их поддерживает **текущая модель** (зависит от провайдера).

В дополнении **нет** собственного проверщика обновлений. **Обновления** устанавливаются через **официальный Магазин дополнений NVDA**, если вы ставили дополнение оттуда. Если вы устанавливаете вручную со [страницы релизов](https://github.com/aaclause/nvda-OpenAI/releases), новые сборки `.nvda-addon` устанавливаются тем же способом.

## Поддерживаемые провайдеры

Настройте **одного или нескольких провайдеров** в NVDA: **Параметры -> Настройки -> AI Hub**. Каждый провайдер может хранить **несколько именованных аккаунтов** (API-ключи, а также при необходимости organization или base URL).

| Провайдер | Роль |
|----------|------|
| [OpenAI](https://platform.openai.com/) | GPT и связанные модели; официальные инструменты транскрибации и TTS |
| [DeepSeek](https://www.deepseek.com/) | API DeepSeek (совместимый с OpenAI) |
| **Custom OpenAI** | Любой OpenAI-совместимый HTTP API (собственный base URL + ключ) |
| **Ollama** | Локальные модели через OpenAI-совместимый endpoint; инструмент управления моделями |
| [Mistral AI](https://mistral.ai/) | Mistral / Pixtral; инструменты Voxtral TTS, OCR и speech-to-text |
| [OpenRouter](https://openrouter.ai/) | Множество сторонних моделей за одним ключом |
| [Anthropic](https://www.anthropic.com/) | Claude |
| [xAI](https://x.ai/) | Grok |
| [Google](https://ai.google.dev/) | Gemini; инструмент Lyria 3 Pro |

Дополнение может использовать API-ключи из **переменных окружения**, если они заданы (например, `OPENAI_API_KEY`, `DEEPSEEK_API_KEY`, `MISTRAL_API_KEY`, `OPENROUTER_API_KEY`, `ANTHROPIC_API_KEY`, `XAI_API_KEY`, `GEMINI_API_KEY` / `GOOGLE_API_KEY` и др.). Основным местом управления аккаунтами остается интерфейс настроек.

### Бэкенды speech-to-text (транскрибации)

Для **транскрибации с микрофона/файла** в основном сценарии (не отдельный инструмент транскрибации OpenAI) можно выбрать **whisper_cpp** (локально), **openai** (Whisper API) и **mistral** в разделе **Audio** настроек AI Hub.

## Установка

1. Откройте [страницу релизов дополнения](https://github.com/aaclause/nvda-OpenAI/releases).
2. Скачайте последний пакет `.nvda-addon`.

## Первоначальная настройка

1. Откройте **NVDA -> Параметры -> Настройки**.
2. Выберите категорию **AI Hub**.
3. В разделе **API Accounts** выберите **Add account...**.
4. В окне аккаунта выберите провайдера, укажите имя аккаунта и заполните обязательные поля (API-ключ для большинства провайдеров; base URL для **Custom OpenAI** и **Ollama**, при этом Ollama по умолчанию использует локальный http://127.0.0.1:11434/v1, если поле пустое).
5. Сохраните, затем при необходимости добавьте другие аккаунты, измените существующие или удалите неиспользуемые из списка.
6. При необходимости настройте **Audio**, **Chat feedback**, **Advanced** / температуру и **Auto-save conversation** (включено по умолчанию).

Пока не настроен хотя бы один аккаунт провайдера, при открытии главного диалога дополнение предложит добавить ключи в настройках AI Hub.

## Обновление со старых сборок “Open AI”

Если вы использовали старую версию этого дополнения:

- **Настройки** мигрируют из устаревшего раздела **`OpenAI`** в **`AIHub`**. Ваши параметры не должны потеряться.
- **Файлы данных** (разговоры, хранилище ключей, вложения) мигрируют из папки **`openai`** в каталоге пользовательской конфигурации NVDA в папку **`aihub`**.

Вручную переносить файлы не нужно, если только у вас нет нестандартной конфигурации.

## Меню NVDA: AI Hub

В меню NVDA вы найдете раздел **AI Hub** (с установленной версией в подписи). Пункты включают:

- **Documentation** — открывает руководство пользователя в браузере (`doc\en\readme.html`).
- **Main dialog...** — открыть окно чата (`NVDA+G` по умолчанию).
- **Conversation history...** — управление сохраненными чатами.
- **Tools** — подменю, группирующее утилиты **OpenAI**, **Mistral**, **Google** и **Ollama** (см. ниже).
- **GitHub repository** / **BasiliskLLM** - быстрые ссылки.

## Главное окно

Открывается через **`NVDA+G`** или пункт **Main dialog...** в меню AI Hub.

### Что можно делать

- Общаться с выбранной моделью; просматривать **Messages** с клавиатурной навигацией и через контекстные меню (например, **j** / **k** для перехода между сообщениями, когда фокус в области сообщений — подсказки для активного поля смотрите на экране).
- Прикреплять **локальные изображения или документы** и добавлять **URL файлов**, когда это поддерживается провайдером. Для неподдерживаемых типов у текущего провайдера перед отправкой может появиться предупреждение.
- Использовать **Paste (file or text)** из контекстного меню поля промпта или **`Ctrl+V`** в поле промпта: дополнение может прикреплять файлы, вставлять текстовые пути или обрабатывать один URL как вложение, когда это уместно.
- Записывать **аудиофрагменты**, прикреплять аудиофайлы и использовать **TTS** для текста промпта, если модель это поддерживает.
- **`Escape`** закрывает главное окно (если не открыто блокирующее модальное окно).
- **`Ctrl+R`** переключает запись с микрофона (когда доступно).
- **`F2`** переименовывает текущий сохраненный разговор (после того как он существует в хранилище).
- **`Ctrl+N`** открывает **новый** экземпляр главного диалога (сессию).

### Опции, зависящие от модели

Некоторые элементы управления появляются или применяются только для определенных моделей:

- **Reasoning** (“thinking”) для моделей, которые это поддерживают; поток рассуждений хранится отдельно от видимого ответа, когда API предоставляет такое разделение.
- **Reasoning effort** и связанные элементы управления там, где провайдер поддерживает уровни.
- **Web search** только для моделей, которые заявляют поддержку веб-поиска.

Точная доступность меняется по мере обновления API провайдеров; интерфейс отражает **текущую выбранную модель**.

### Системный промпт

Системный промпт задает поведение модели. По умолчанию предоставлен вариант, подходящий для задач доступности; вы можете его редактировать, сбрасывать из контекстного меню и при желании сохранять последний использованный вариант (настраивается в параметрах). Системный промпт расходует токены так же, как и любой другой ввод.

## История разговоров

Используйте **Conversation history...** в меню AI Hub или назначьте жест в **Жесты ввода -> AI Hub**.

Вы можете просматривать, открывать, переименовывать, удалять и создавать разговоры. В главном диалоге **F2** и **Ctrl+N** помогают управлять текущей сессией.

### Автосохранение

Если в настройках включено **Auto-save conversation** (по умолчанию), дополнение сохраняет (или обновляет) сохраненный разговор **после каждого завершенного ответа ассистента**, а также может сохранять состояние при закрытии диалога, если есть что сохранить. Также можно сохранять разговор из контекстного меню поля **Messages**. Если автосохранение выключено, используйте ручное сохранение, когда хотите сохранить разговор.

## Задать вопрос (голос)

У этой команды **нет жеста по умолчанию**. Назначьте его в **Жесты ввода -> AI Hub**.

- Первое нажатие: начать запись.
- Второе нажатие во время записи: остановить и отправить.
- Если ответ воспроизводится как аудио, нажмите еще раз, чтобы остановить воспроизведение.

**Режимы:**

- **Direct audio** — если выбранная модель поддерживает аудиоввод, запись может отправляться как аудио без отдельного шага транскрибации.
- **Transcribe then chat** — в противном случае выбранный бэкенд транскрибации обрабатывает запись, после чего текст отправляется чат-модели.

Если в фокусе главный диалог, используется его **текущая модель**; иначе дополнение выбирает подходящую модель среди настроенных провайдеров.

## Подменю Tools

Пункт **Tools** в меню AI Hub открывает диалоги, сгруппированные по провайдерам (для каждого может потребоваться соответствующий API-аккаунт):

| Раздел меню | Инструмент |
|-----------|------|
| Mistral | **Voxtral TTS...**, **OCR...**, **Speech to Text...** |
| Google | **Lyria 3 Pro...** |
| OpenAI | **TTS...**, **Transcription / Translation...** |
| Ollama | **Model manager...** |

Если для провайдера инструмента не настроен аккаунт, дополнение сообщит добавить его в настройках AI Hub.

## Глобальные команды

Все жесты по умолчанию можно изменить в **NVDA -> Параметры -> Жесты ввода -> AI Hub**.

| Жест | Действие |
|---------|--------|
| `NVDA+G` | Показать главное окно AI Hub |
| `NVDA+E` | Сделать скриншот и описать (добавляет изображение в сессию) |
| `NVDA+O` | Описать область текущего объекта навигатора |
| *(нет жеста по умолчанию)* | История разговоров. Назначьте в Жесты ввода -> AI Hub. |
| *(нет жеста по умолчанию)* | Задать вопрос (запись / отправка / остановка аудио). Назначьте в Жесты ввода -> AI Hub. |
| *(нет жеста по умолчанию)* | Переключить запись с микрофона и транскрибацию. Назначьте в Жесты ввода -> AI Hub. |

## Где хранятся данные

Рабочие файлы, индекс сохраненных разговоров, единый `accounts.json` и вложения находятся в каталоге пользовательской конфигурации NVDA, в папке **`aihub`** (после миграции из `openai`). Временные файлы используют подпапку `tmp` и очищаются при необходимости (например, при завершении дополнения или закрытии диалога).

## Обязательные зависимости (автозагрузка во время сборки)

Сборка через `scons` размещает runtime-библиотеки в:

`addon/globalPlugins/AIHub/libs/`

Если обязательной библиотеки нет, `scons` скачивает зафиксированные wheel-пакеты и извлекает только нужное в эту папку. Текущие зафиксированные зависимости:

- **[markdown-it-py](https://pypi.org/project/markdown-it-py/)** `3.0.0` + **[mdurl](https://pypi.org/project/mdurl/)** `0.1.2` — извлекается в `libs/` для рендеринга Markdown в чате.
- **[Pillow](https://pypi.org/project/Pillow/)** `12.1.1` для:
  - Python `3.11` `win32` -> `libs/lib_py3.11_win32/`
  - Python `3.13` `win_amd64` -> `libs/lib_py3.13/`

Каталог `libs` намеренно игнорируется в git; контрибьюторам не нужно коммитить вендорные артефакты.

## Устранение неполадок (кратко)

- **“No account configured”** — добавьте API-ключ для выбранного провайдера в настройках **AI Hub**.
- **Провайдер отклоняет вложение** — проверьте тип и размер файла; попробуйте другую модель или провайдера, поддерживающего нужный медиаформат.

Для сообщений об ошибках и вкладов используйте ссылку **GitHub repository** в меню AI Hub.
