אם אתם רוצים חוויית שולחן עבודה ייעודית עם תהליכי עבודה נוספים, ראו את [BasiliskLLM](https://github.com/SigmaNight/basiliskLLM/) (אפליקציה עצמאית לצד תוסף NVDA מינימלי). ‏AI-Hub נשאר אפשרות מלאה בתוך NVDA.

# AI-Hub

**AI-Hub** הוא תוסף ל-NVDA שמחבר את קורא המסך שלכם לכמה ממשקי API של מודלי שפה גדולים (LLM). אפשר להשתמש בו לכתיבה, סיכום, סיוע בתרגום, ראייה (תמונות וצילומי מסך), שאלות קוליות, תמלול ותיבות דו-שיח אופציונליות של כלים (TTS, OCR ועוד) — בלי לצאת מ-NVDA.

**שם החבילה** של התוסף ב-NVDA עדיין `openai` (לתאימות עם התקנות קיימות). **שם התצוגה** שתראו בתפריטים ובהגדרות הוא **AI-Hub**.

## תכונות במבט מהיר

- **צ'אט** בתיבת דו-שיח ראשית ייעודית עם היסטוריה, הנחיית מערכת ובחירת מודל/חשבון.
- **תמונות ומסמכים** כקבצים מצורפים מתוך קבצים; **כתובות URL** לקבצים מרוחקים עם בדיקות סוג המותאמות ל-**ספק הנבחר**.
- **הדבקה חכמה** בשדה ההנחיה: הדבקת קבצים מלוח הגזירים, נתיבים מתוך טקסט או כתובת URL בודדת (זמין גם מתפריט ההקשר של ההנחיה). ‏`Ctrl+V` משתמש באותה לוגיקה כשהמיקוד בשדה ההנחיה.
- **שמירת שיחות והיסטוריה** עם שינוי שם, מחיקה ופתיחה מחדש.
- **שאל שאלה** מכל מקום (ללא מקש ברירת מחדל): הקצו מחווה ב-**Input Gestures → AI-Hub** כדי להקליט, לשלוח ולשמוע או לקרוא את התשובה.
- **תיאור גלובלי**: צילום מסך (`NVDA+E`) או אזור אובייקט הניווט (`NVDA+O`) נשלחים לסשן צ'אט.
- תת-תפריט **Tools** (תחת NVDA → AI-Hub): כלי עזר לפי ספק, כגון TTS, OCR, דיבור-לטקסט, אודיו Lyria וניהול מודלים של Ollama.
- אפשרויות **Reasoning / Web search** מופיעות רק כאשר ה-**מודל הנוכחי** תומך בהן (משתנה לפי ספק).

תוסף זה **לא** כולל בודק עדכונים מובנה משלו. **עדכונים** מנוהלים דרך **חנות התוספים הרשמית של NVDA** כשמתקינים משם. אם מתקינים ידנית מתוך [עמוד הגרסאות](https://github.com/aaclause/nvda-OpenAI/releases), מתקינים גרסאות `.nvda-addon` חדשות יותר באותה דרך.

## ספקים נתמכים

הגדירו **ספק אחד או יותר** ב-NVDA תחת **Preferences → Settings → AI-Hub**. כל ספק יכול להכיל **כמה חשבונות עם שם** (מפתחות API, וכן ארגון אופציונלי או כתובת בסיס URL כשזה רלוונטי).

| ספק | תפקיד |
|----------|------|
| [OpenAI](https://platform.openai.com/) | GPT ומודלים קשורים; תיבות דו-שיח רשמיות לכלי תמלול ו-TTS |
| [DeepSeek](https://www.deepseek.com/) | ממשק DeepSeek (תואם OpenAI) |
| **Custom OpenAI** | כל API מסוג HTTP שתואם OpenAI (כתובת בסיס מותאמת + מפתח) |
| **Ollama** | מודלים מקומיים דרך נקודת קצה תואמת OpenAI; כלי ניהול מודלים |
| [Mistral AI](https://mistral.ai/) | Mistral / Pixtral; כלי Voxtral TTS, OCR ודיבור-לטקסט |
| [OpenRouter](https://openrouter.ai/) | מודלים רבים של צד שלישי מאחורי מפתח אחד |
| [Anthropic](https://www.anthropic.com/) | Claude |
| [xAI](https://x.ai/) | Grok |
| [Google](https://ai.google.dev/) | Gemini; כלי Lyria 3 Pro |

התוסף יכול לקלוט מפתחות API מתוך **משתני סביבה** כשהם מוגדרים (למשל `OPENAI_API_KEY`, `DEEPSEEK_API_KEY`, `MISTRAL_API_KEY`, `OPENROUTER_API_KEY`, `ANTHROPIC_API_KEY`, `XAI_API_KEY`, `GEMINI_API_KEY` / `GOOGLE_API_KEY` ועוד). ממשק ההגדרות נשאר המקום המרכזי לניהול חשבונות.

### מנועי דיבור-לטקסט (תמלול)

עבור **תמלול ממיקרופון / קובץ** בתוך הזרימה הראשית (ולא בכלי התמלול הנפרד של OpenAI), אפשר לבחור בין **whisper_cpp** (מקומי), **openai** (ממשק Whisper API) ו-**mistral**, תחת סעיף **Audio** בהגדרות AI-Hub.

## התקנה

1. פתחו את [עמוד גרסאות התוסף](https://github.com/aaclause/nvda-OpenAI/releases).
2. הורידו את חבילת `.nvda-addon` העדכנית ביותר.

## הגדרה ראשונית

1. פתחו **NVDA → Preferences → Settings**.
2. בחרו את קטגוריית **AI-Hub**.
3. ב-**API Accounts** בחרו **Add account...**.
4. בחלון החשבון בחרו ספק, הזינו שם חשבון ומלאו את השדות הנדרשים (מפתח API לרוב הספקים; כתובת בסיס עבור **Custom OpenAI** ו-**Ollama**, וב-Ollama ברירת המחדל היא http://127.0.0.1:11434/v1 כאשר השדה ריק).
5. שמרו, ולאחר מכן ניתן להוסיף חשבונות נוספים, לערוך חשבונות קיימים או להסיר חשבונות לא בשימוש.
6. ניתן גם להתאים הגדרות **Audio**, ‏**Chat feedback**, ‏**Advanced** / טמפרטורה ו-**Auto-save conversation** (מופעל כברירת מחדל).

עד שיוגדר לפחות חשבון ספק אחד, פתיחת תיבת הדו-שיח הראשית תבקש מכם להוסיף מפתחות בהגדרות AI-Hub.

## שדרוג מבניות “Open AI” ישנות יותר

אם השתמשתם בגרסה ישנה יותר של התוסף הזה:

- **הגדרות** מועברות מסעיף התצורה הישן **`OpenAI`** אל **`AIHub`**. ההעדפות שלכם לא אמורות ללכת לאיבוד.
- **קובצי נתונים** (שיחות, מאגר מפתחות, קבצים מצורפים) מועברים מהתיקייה **`openai`** תחת ספריית תצורת המשתמש של NVDA אל **`aihub`**.

אין צורך להעביר קבצים ידנית, אלא אם אתם משתמשים בהגדרה מותאמת אישית.

## תפריט NVDA: AI-Hub

בתפריט NVDA תמצאו את **AI-Hub** (עם הגרסה המותקנת בתווית). הפריטים כוללים:

- **Documentation** — פותח את מדריך המשתמש בדפדפן שלכם (`doc\en\readme.html`).
- **Main dialog…** — פתיחת חלון הצ'אט (`NVDA+G` כברירת מחדל).
- **Conversation history…** — ניהול שיחות שמורות.
- **Tools** — תת-תפריט שמקבץ כלי עזר של **OpenAI**, ‏**Mistral**, ‏**Google** ו-**Ollama** (ראו למטה).
- **GitHub repository** / **BasiliskLLM** - קישורים מהירים.

## תיבת הדו-שיח הראשית

פתחו עם **`NVDA+G`** או דרך **Main dialog…** מתפריט AI-Hub.

### מה אפשר לעשות

- לשוחח עם המודל הנבחר; לעבור על **Messages** עם ניווט מקלדת ותפריטי הקשר (למשל **j** / **k** למעבר בין הודעות כשהמיקוד באזור ההודעות — ראו רמזים על המסך לשדה הפעיל).
- לצרף **תמונות או מסמכים מקומיים** ולהוסיף **כתובות URL לקבצים** היכן שהספק תומך בכך. ייתכן שתתקבל אזהרה על סוגים שאינם נתמכים עבור הספק הנוכחי לפני שליחה.
- לבצע **Paste (file or text)** מתפריט ההקשר של ההנחיה, או **`Ctrl+V`** בתוך ההנחיה: התוסף עשוי לצרף קבצים, להכניס נתיבי טקסט או להתייחס ל-URL בודד כקובץ מצורף כשזה מתאים.
- להקליט קטעי **אודיו**, לצרף קובצי אודיו, ולהשתמש ב-**TTS** עבור טקסט ההנחיה כאשר המודל תומך בכך.
- **`Escape`** סוגר את תיבת הדו-שיח הראשית (כשאין חלון מודאלי חוסם פתוח).
- **`Ctrl+R`** מפעיל/מכבה הקלטת מיקרופון (כשזה רלוונטי).
- **`F2`** משנה את שם השיחה השמורה הנוכחית (לאחר שהיא כבר קיימת באחסון).
- **`Ctrl+N`** פותח מופע חדש של תיבת הדו-שיח הראשית (**new** session).

### אפשרויות שתלויות במודל

חלק מהפקדים מופיעים או חלים רק עבור מודלים מסוימים:

- **Reasoning** (“thinking”) עבור מודלים שחושפים זאת; reasoning מוזרם נשמר בנפרד מהתשובה הגלויה כשה-API מספק הבחנה כזו.
- **Reasoning effort** ופקדים קשורים במקומות שבהם הספק תומך ברמות.
- **Web search** רק עבור מודלים שמצהירים על תמיכה בחיפוש ברשת.

הזמינות המדויקת משתנה כשספקים מעדכנים את ממשקי ה-API שלהם; ממשק המשתמש משקף את ה-**מודל הנבחר כעת**.

### הנחיית מערכת

הנחיית המערכת מכוונת את התנהגות המודל. קיימת ברירת מחדל המתאימה לסיוע בנגישות; תוכלו לערוך אותה, לאפס אותה מתפריט ההקשר, ואופציונלית לשמור את ההנחיה האחרונה שבה השתמשתם (ניתן להגדרה בהגדרות). הנחיית המערכת צורכת טוקנים כמו כל קלט אחר.

## היסטוריית שיחות

השתמשו ב-**Conversation history…** מתפריט AI-Hub, או הקצו מחווה תחת **Input Gestures → AI-Hub**.

אפשר להציג רשימה, לפתוח, לשנות שם, למחוק וליצור שיחות. מתיבת הדו-שיח הראשית, ‏**F2** ו-**Ctrl+N** מסייעים בניהול הסשן הנוכחי.

### שמירה אוטומטית

אם האפשרות **Auto-save conversation** מופעלת בהגדרות (ברירת מחדל), התוסף שומר (או מעדכן) את השיחה המאוחסנת **לאחר כל תגובת עוזר שהושלמה**, ועשוי לשמור מצב כשסוגרים את תיבת הדו-שיח אם יש מה לשמור. אפשר גם לשמור מתפריט ההקשר של השדה **Messages**. אם שמירה אוטומטית כבויה, השתמשו בשמירה ידנית כשרוצים לשמור את המידע לאורך זמן.

## שאל שאלה (קול)

לפקודה זו **אין מקש ברירת מחדל**. הקצו אחד תחת **Input Gestures → AI-Hub**.

- לחיצה ראשונה: התחלת הקלטה.
- לחיצה שנייה בזמן הקלטה: עצירה ושליחה.
- אם התשובה מושמעת כאודיו, לחצו שוב כדי לעצור את ההשמעה.

**מצבים:**

- **Direct audio** — אם המודל הנבחר תומך בקלט אודיו, אפשר לשלוח את ההקלטה שלכם כאודיו ללא שלב תמלול נפרד.
- **Transcribe then chat** — אחרת, מנוע התמלול המוגדר מעבד את ההקלטה ואז הטקסט נשלח למודל הצ'אט.

אם תיבת הדו-שיח הראשית ממוקדת, נעשה שימוש ב-**מודל הנוכחי** שלה; אחרת התוסף בוחר מודל מתאים מבין הספקים המוגדרים.

## תת-תפריט Tools

הפריט **Tools** תחת תפריט AI-Hub פותח תיבות דו-שיח מקובצות לפי ספק (כל אחת עשויה לדרוש את חשבון ה-API המתאים):

| אזור תפריט | כלי |
|-----------|------|
| Mistral | **Voxtral TTS…**, **OCR…**, **Speech to Text…** |
| Google | **Lyria 3 Pro…** |
| OpenAI | **TTS…**, **Transcription / Translation…** |
| Ollama | **Model manager…** |

אם לא מוגדר חשבון עבור ספק הכלי, התוסף ינחה אתכם להוסיף אחד בהגדרות AI-Hub.

## פקודות גלובליות

ניתן לשנות את כל מחוות ברירת המחדל ב-**NVDA → Preferences → Input Gestures → AI-Hub**.

| מחווה | פעולה |
|---------|--------|
| `NVDA+G` | הצגת תיבת הדו-שיח הראשית של AI-Hub |
| `NVDA+E` | צילום מסך ותיאור (מוסיף תמונה לסשן) |
| `NVDA+O` | תיאור אזור אובייקט הניווט הנוכחי |
| *(אין מחוות ברירת מחדל)* | היסטוריית שיחות. הקצו ב-Input Gestures → AI-Hub. |
| *(אין מחוות ברירת מחדל)* | שאל שאלה (הקלטה / שליחה / עצירת אודיו). הקצו ב-Input Gestures → AI-Hub. |
| *(אין מחוות ברירת מחדל)* | הפעל/כבה הקלטת מיקרופון ותמלול. הקצו ב-Input Gestures → AI-Hub. |

## היכן הנתונים נשמרים

קובצי עבודה, אינדקס שיחות שמורות, ‏`accounts.json` מאוחד וקבצים מצורפים נשמרים תחת ספריית **תצורת המשתמש** של NVDA, בתיקיית **`aihub`** (לאחר מעבר מ-`openai`). קבצים זמניים משתמשים בתת-תיקייה `tmp` ומנוקים כשזה סביר (למשל בסיום התוסף או בסגירת תיבת הדו-שיח).

## תלויות נדרשות (נשלפות אוטומטית בזמן build)

תהליכי build משתמשים ב-`scons` כדי לאכלס ספריות ריצה תחת:

`addon/globalPlugins/AIHub/libs/`

כאשר ספרייה נדרשת חסרה, `scons` מוריד חבילות wheel מוצמדות גרסה ומחלץ רק את מה שנדרש לתיקייה זו. התלויות המוצמדות הנוכחיות הן:

- **[markdown-it-py](https://pypi.org/project/markdown-it-py/)** `3.0.0` + **[mdurl](https://pypi.org/project/mdurl/)** `0.1.2` — מחולץ אל `libs/` עבור רינדור Markdown בצ'אט.
- **[Pillow](https://pypi.org/project/Pillow/)** `12.1.1` עבור:
  - Python `3.11` `win32` -> `libs/lib_py3.11_win32/`
  - Python `3.13` `win_amd64` -> `libs/lib_py3.13/`

ספריית `libs` מוגדרת במכוון להתעלמות ב-git; תורמים אינם צריכים לבצע commit לארטיפקטים vendored.

## פתרון תקלות (קצר)

- **"No account configured"** — הוסיפו מפתח API עבור הספק שבחרתם בהגדרות **AI-Hub**.
- **Provider rejects an attachment** — בדקו את סוג וגודל הקובץ; נסו מודל או ספק אחר שתומך במדיה שאתם צריכים.

לבעיות ותרומות, השתמשו בקישור **GitHub repository** מתפריט AI-Hub.
